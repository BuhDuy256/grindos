## Bug Found

### BUG-08: Judgment Day (Day 30) generates multiple tasks instead of ONE
**Severity:** Medium (spec violation)  
**Spec requirement (`04_batch_execution_logic.md §2.A`):** "If day equals 30, activate Judgment Day protocol: swap standard task array for exactly ONE comprehensive evaluation challenge task."  
**Actual:** Day 30 generated 3 tasks (5m + 80m + 80m = 165 mins).  
**Root cause:** The Judgment Day instruction is in the `{phase_instruction}` placeholder of `prompt.txt`. The model interpreted it as a guideline, not a hard constraint. Pydantic `response_schema=DailyPlanOutput` only validates `List[TaskOutput]` — it does not constrain list length to 1.  
**Fix options:**
1. Add a `max_tasks: int` field to `DailyPlanOutput` or validate `len(tasks) == 1` in `validate_plan_output()` when `arc_day_index == 30`.
2. Use a separate `JudgmentDayOutput(BaseModel)` with `task: TaskOutput` (singular) passed as `response_schema` specifically on Day 30.
3. Make `validate_plan_output` raise if `arc_day_index == 30 and len(plan.tasks) != 1`.  
**Status:** ⚠️ Open — recommended fix is Option 2 (separate schema for Day 30).
